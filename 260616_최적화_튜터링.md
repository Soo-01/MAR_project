> 목적: AI가 대신 짜준 코드를 "이해"하는 수준을 넘어, **내가 직접 cost를 설계하고 solver를 돌릴 수 있게** 되는 것.
> 방식: 소크라테스식. 묻고 → 내가 답하고 → 이론(수식)과 실무(코드)로 매번 검증. 정답을 먼저 받지 않는다.

## 학습 규칙 (우리가 지킬 것)

1. **내가 먼저 답한다.** 튜터(Claude)는 질문을 던지고, 내가 시도하기 전에는 정답 코드를 주지 않는다.
2. **틀려도 적는다.** 틀린 시도도 로그에 남긴다. 어디서 틀렸는지가 학습의 핵심.
3. **매 개념마다 두 가지 테스트:**
   - 이론 테스트: 수식/논리로 "왜"를 설명한다.
   - 코드 테스트: numpy로 직접 구현하고, 검증 스크립트로 숫자 확인한다.
4. **검증은 말이 아니라 숫자로.** "맞는 것 같아요"가 아니라 finite-difference, known-value 체크로 통과/실패를 본다.
5. 진도는 천천히. 한 번에 한 개념.

# 1교시 — cost function이란 무엇인가

## 1.1 핵심 개념 (튜터 설명)

최적화의 출발점은 단 한 문장이다:

> **cost function $J(q)$ = "지금 자세 $q$가 얼마나 나쁜가"를 하나의 실수(scalar)로 매긴 점수.**

solver가 하는 일은 단순하다. 이 점수를 **가장 작게** 만드는 $q$를 찾는다. $\min_q J(q)$.

그래서 최적화에서 "내가 하는 일"의 90%는 사실 solver를 돌리는 게 아니라,
**말로 된 목표("숟가락을 수평으로 유지해라")를 $J(q)$라는 함수로 번역하는 것**이다.
이 번역을 잘하면 solver는 알아서 풀고, 못하면 solver가 엉뚱한 데로 간다.

## 1.2 좋은 cost가 갖춰야 할 3조건

번역이 "잘 된" cost는 다음 셋을 만족한다. (지금은 외우지 말고 감만 잡아둘 것)

1. **정렬성 (correctness):** 목표가 완벽히 달성됐을 때 $J$가 최소(보통 0)이고, 목표에서 멀어질수록 커진다.
2. **매끄러움 (smoothness):** $q$에 대해 미분 가능하고, 갑자기 꺾이거나(non-smooth) 평평한(flat) 구간이 없어야 한다. → gradient 기반 solver가 길을 찾을 수 있어야 하니까.
3. **스케일 (scale):** 단위와 크기가 적절해서, 나중에 여러 항을 weight로 합칠 때 한 항이 다른 항을 압도하지 않는다.

## 1.3 [1번 문제] — 첫 번역 연습 (← 내 차례)

상황: 숟가락의 현재 법선 벡터(world frame, 단위벡터)를 $n(q)$라 하자.
우리가 원하는 "수평" 목표 방향은 $n_{ref} = [0, 0, 1]^T$ (위쪽).

**Q1-1 (이론):** $n(q)$ 와 $n_{ref}$ 만 가지고, 위 "3조건"을 의식하면서

(a) 완벽 수평일 때 값이 얼마인지,
(b) 90도로 완전히 눕혔을 때 값이 얼마인지
