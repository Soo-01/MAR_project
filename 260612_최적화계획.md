# 식사 보조 로봇의 Phase-Aware Trajectory Optimization 프레임워크

## 1. 시스템 아키텍처 개요 (System Architecture)

본 프로젝트는 기존의 단편적인 IK(Inverse Kinematics) 풀이 방식에서 벗어나, 전체 동작의 안전성과 부드러움을 보장하기 위해 시스템을 다음과 같이 5단계의 계층 구조로 분리하여 재정의한다. 최적화(Optimization)는 제어기(Controller)를 대체하는 것이 아니라, 제어기가 추종할 최적의 레퍼런스 경로를 생성하는 역할을 한다.

```text
[Task Planner] ──> [Initial Trajectory Generator] ──> [Trajectory Optimizer]
                                                               │ (q_ref, dq_ref)
[Runtime Monitor] <── [Robot / Simulation] <── [Tracking Controller (PD)]
```

* **Modeling**: 로봇, 숟가락(End-Effector), 환경(식판, 음식, 입) 및 제약 조건을 수학적으로 정의
* **Task Planning**: 고수준(High-level) 단계에서 식사를 위한 4가지 Phase 시퀀스 결정
* **Trajectory Optimization**: 각 Phase의 목적에 맞는 비용 함수(Cost)를 기반으로 최적의 관절 경로 $q_{0:N}$ 계산
* **Tracking Control (PD)**: 계산된 $q_{ref}(t)$를 로봇이 물리적으로 정확하게 따라가도록 모터 명령 출력
* **Safety / Runtime Monitor**: 실행 중 실시간 오차, 예기치 못한 충돌, 특이점 발생을 감시하고 비상 정지 및 Re-planning 트리거

## 2. 수학적 모델링 (Modeling)

### 2.1 로봇 및 숟가락 모델 (Robot & Spoon Model)

* **관절 공간 상태**: $q \in \mathbb{R}^n$
* **정기구학 및 자코비안**: $x = FK(q), \quad J(q) = \frac{\partial x}{\partial q}$
* **숟가락 고유 벡터**:
  * $p_{\text{tip}}(q)$: 숟가락 끝단 위치 (음식을 뜨고 입에 넣는 기준점)
  * $n_{\text{spoon}}(q)$: 숟가락 bowl의 법선 벡터 (수평 유지 및 국물 넘침 방지 기준)
  * $f_{\text{spoon}}(q)$: 숟가락의 진행 방향 벡터 (Yaw 제어 및 음식 접근 기준)
* **Head-Down Metric ($h(q)$)**: 음식을 뜨기 위해 숟가락 머리가 손목보다 아래를 향하는 정도
  $$h(q) = z_{\text{wrist}}(q) - z_{\text{head}}(q) \quad (h(q) > 0 \text{ 일 때 Head-down 상태})$$

### 2.2 환경 모델 (Environment Model)

* $p_{\text{food}}$: 음식의 3차원 위치
* $p_{\text{mouth}}$: 사용자의 입 위치
* $z_{\text{tray}}$: 식판 바닥의 임계 높이 (지면 제약)

## 3. 식사 동작의 4단계 Phase 정의 (Task Planning)

식사 보조 로봇은 각 구간마다 달성해야 하는 물리적 목표와 제한 조건이 상이하므로, Phase-Aware Cost를 적용한다.

| Phase 명칭 | 주요 작업 (Task) | 핵심 최적화 가중치 (Key Weights) |
| --- | --- | --- |
| 1. Pre-scoop (접근) | 음식 위치 위로 숟가락을 이동 | 위치 추종 ($w_p$), Head-down 유도 ($w_h$) |
| 2. Engage / Drag (뜨기) | 식판 바닥을 긁으며 음식을 담음 | 경로 추종, 진행 방향 정렬 ($w_f$), 식판 충돌 회피 |
| 3. Lift / Carry (들기) | 음식을 담은 채 위로 들어올림 | 숟가락 수평 유지 ($w_n$), 급격한 가속도 제한 |
| 4. Mouth Delivery (전달) | 사용자 입 앞까지 안전하게 이동 | 최고 수준의 수평 유지 ($w_n$), 특이점 회피 ($w_s$), 안전 거리 확보 |

## 4. 궤적 최적화 문제 재구성 (Trajectory Optimization)

### 4.1 Decision Variable

동역학을 배제한 기구학적 궤적 최적화(Kinematic Trajectory Optimization) 단계에서는 전체 타임스텝 $N$ 동안의 관절각 시퀀스를 하나의 변수로 묶어 최적화한다.
$$X = q_{0:N} = [q_0, q_1, \dots, q_N]$$

### 4.2 전체 비용 함수 (Total Cost Function)

$$\min_{q_{0:N}} J(q_{0:N}) = J_{\text{task}} + J_{\text{posture}} + J_{\text{safety}} + J_{\text{smooth}}$$

**① Task Tracking Cost**
숟가락 끝단($p_{\text{tip}}$)이 각 Phase별 레퍼런스 경로를 정밀하게 추종하도록 유도한다.
$$J_{\text{task}} = \sum_{k=0}^{N} w_p(k) \|p_{\text{tip}}(q_k) - p_k^{ref}\|^2$$

**② Spoon Posture Cost**
숟가락의 수평 유지, 진행 방향 정렬, Head-down 모션을 제어한다. 외적($\times$)을 활용하여 타겟 방향과의 정렬 오차를 최소화한다.
$$J_{\text{posture}} = \sum_{k=0}^{N} \left[ w_n(k)\|n_{\text{spoon}}(q_k) \times n_k^{ref}\|^2 + w_f(k)\|f_{\text{spoon}}(q_k) \times f_k^{ref}\|^2 + w_h(k)\max(0, h_k^{ref} - h(q_k))^2 \right]$$

**③ Safety Cost**
관절 한계 영역, 특이점(Singularity) 근처, 환경과의 충돌 예상 지점에 패널티(Penalty)를 부여한다.
$$J_{\text{safety}} = \sum_{k=0}^{N} \left[ w_l \phi_{\text{limit}}(q_k) + w_s \phi_{\text{sing}}(q_k) + w_c \phi_{\text{collision}}(q_k) \right]$$

* **관절 한계 패널티 ($\phi_{\text{limit}}$)**: 마진($m_i$) 이내로 관절이 한계에 다다르면 비용 급증
  $$\phi_{\text{limit}}(q) = \sum_i \left[ \max(0, q_i - (q_i^{\max} - m_i))^2 + \max(0, (q_i^{\min} + m_i) - q_i)^2 \right]$$
* **특이점 회피 패널티 ($\phi_{\text{sing}}$)**: 자코비안의 최소 특이값($\sigma_{\min}$)이 임계치($\epsilon$)보다 작아지면 비용 부여
  $$\phi_{\text{sing}}(q) = \max(0, \epsilon - \sigma_{\min}(J(q)))^2$$

**④ Smoothness Cost**
제어기(PD)가 무리 없이 추종할 수 있도록 관절의 속도와 가속도(Jerk 감쇄 효과)를 최소화하여 부드러운 움직임을 만든다.
$$J_{\text{smooth}} = \sum_{k=1}^{N} w_v \|q_k - q_{k-1}\|^2 + \sum_{k=1}^{N-1} w_a \|q_{k+1} - 2q_k + q_{k-1}\|^2$$

## 5. 제약 조건의 분리 (Hard Constraints vs Soft Costs)

수학적 최적화의 안정성과 물리적 안전을 보장하기 위해 제약 조건을 엄격하게 이원화한다.

**Hard Constraints (절대 깨져서는 안 되는 규칙)**

* 물리적 관절 가동 범위: $q_{\min} \le q_k \le q_{\max}$
* 자코비안 가역성 보장: $\sigma_{\min}(J(q_k)) \ge \epsilon$
* 환경 및 자가 충돌 방지: $d_{\text{collision}}(q_k) \ge 0$

**Soft Costs (비용 함수를 통해 최소화할 목표)**

* 경로 추종 오차, 숟가락 기울기 오차, 저크(Jerk) 최소화, 선호 자세 유지

> **💡 현실적 구현 팁**: 초기 유한 시간 내 수렴을 위해 1차 구현 시에는 Hard Constraint를 높은 가중치의 Soft Penalty 형태로 목적함수에 통합한 뒤, 결과물 궤적을 Validation 필터로 걸러내는(Reject) 방식을 추천한다.

## 6. 하위 제어기 설계 (Tracking Control)

최적화 단계에서 도출된 최적의 궤적 $q_{ref}(t)$와 속도 $\dot{q}_{ref}(t)$는 시뮬레이션 환경(MuJoCo) 및 실제 로봇 구동을 위해 PD 제어기로 전달된다.

### 6.1 기본 PD 제어 (Position/Velocity 입력 구조)

$$u(t) = K_p (q_{ref}(t) - q(t)) + K_d (\dot{q}_{ref}(t) - \dot{q}(t))$$

### 6.2 향후 확장: 계산 토크 제어 (Computed Torque Control)

로봇의 비선형 동역학 특성(질량 행렬 $M$, 원심력/전향력 $C$, 중력 $g$)을 상쇄하여 추종 성능을 극대화할 때 적용한다.
$$\tau = M(q)\ddot{q}_{ref} + C(q, \dot{q})\dot{q} + g(q) + K_p e + K_d \dot{e}$$

## 7. 단계별 실전 개발 및 검증 계획

**[1단계] 정량적 비교 Baseline 구축**
개선 전후를 명확하게 비교 평가할 수 있도록 현재 시스템(Multi-start DLS IK + Keyframe 간 Joint Interpolation)의 구동 데이터로부터 다음 메트릭(Metric)을 추출하고 로깅하는 코드를 먼저 완성한다.

* 평가 메트릭: Max Position Error, Max Spoon Tilt, Min Head-down Depth, Min Singular Value ($\sigma_{\min}$), Joint Limit Margin, Total Trajectory Length / Smoothness Cost

**[2단계] 특정 Phase 타겟 궤적 최적화 구현**
전체 시퀀스를 한 번에 풀지 말고, 가장 실패율이 높거나 정밀도가 요구되는 'Drag(뜨기)' 혹은 'Lift to Mouth(이동)' 구간 하나만 타겟으로 잡는다.
기존 Baseline 보간 경로를 최적화의 초기 궤적($q_{0:N}^{init}$)으로 설정한다.
SciPy(SLSQP) 또는 NLopt 등의 라이브러리를 활용하여 위의 4가지 핵심 Cost를 최소화하는 단일 구간 최적화를 수행한다.

**[3단계] 전구간 확장 및 검증 데이터 확보**
단일 구간 성공 시 전체 Phase로 확장하고, 기존 Baseline 대비 "최대 기울어짐 모멘트 감소율", "특이점 근접 빈도 감소율" 등을 데이터 시각화하여 최적화 도입의 당위성을 증명한다.

## 8. 딥러닝(AI) 도입 시 올바른 포지셔닝 (향후 확장성)

딥러닝 모델을 로봇 관절을 직접 움직이는 엔드투엔드(End-to-End) 제어기로 사용하면 제약 조건을 보장할 수 없어 대단히 위험하다. 대신 최적화 연산의 속도를 극대화하는 보조 도구로 포지셔닝한다.

* **AI의 역할**: 음식 및 사용자 위치 정보를 받아, 최적의 수렴 가능성이 높은 초기 고품질 궤적($q_{0:N}^{init}$)을 순식간에 예측 시뮬레이션 지점(Warm-start)으로 제공한다.
* **최적화의 역할**: AI가 준 초기 궤적을 바탕으로 수 밀리초(ms) 내에 물리적 제약 조건(Hard Constraints)을 100% 만족하는 정밀 안전 궤적($q_{0:N}^*$)으로 다듬는다.
